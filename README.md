## 🚀 Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/YOUR_USERNAME/ozon-position-tracker.git
cd ozon-position-tracker
```


### 2. Создать виртуальное окружение
``` python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```
### 3. Установить зависимости
```
pip install -r requirements.txt
```
### 4. Настроить окружение
```
# Windows
copy .env.example .env

# Mac / Linux
cp .env.example .env
```
### 5. Запустить парсер
```
python main.py --query "нож туристический" --sku 1308088830
```

💻 Использование

Одиночный поиск

python main.py --query "нож туристический" --sku 1308088830

python main.py --query "нож туристический" --sku 1308088830 --no-headless

Термос Stanley	Стабильный повседневный спрос — предсказуемая выдача, удобно для тестирования

Почему именно эти товары

Все три товара из категории "туризм и активный отдых" — реальная конкурентная ниша на Ozon.

Выбраны намеренно разные уровни конкуренции:


Высокая конкуренция (нож) — проверяем работу пагинации, товар может быть далеко в выдаче
Средняя конкуренция (рюкзак) — специфичный запрос, товар должен быть ближе к топу
Стабильный спрос (термос) — удобен для теста устойчивости, выдача не меняется быстро

🛡️ Устойчивость (Часть 2)

Запуск теста

python run_stability_test.py

Скрипт выполняет 3 запуска подряд с интервалом 30 секунд

и сохраняет отчёт в 
results/stability_test_{timestamp}.json
.


Что делается чтобы скрипт не заблокировали

Механизм	Реализация
undetected-chromedriver	Патчит ChromeDriver на уровне бинарника — убирает все automation флаги
Ротация User-Agent	Каждый запуск — новый случайный User-Agent (Chrome на Win/Mac/Linux)
Случайные задержки	Нормальное распределение 3–7с (реалистичнее равномерного)
Имитация скроллинга	3–6 шагов с паузами — загружает lazy-load товары как человек
Случайные движения мыши	ActionChains с 2–4 случайными позициями курсора
Реалистичный viewport	Случайный из пула реальных размеров экранов (не стандартный headless)
JS-патчи	Переопределяем 
navigator.webdriver
, 
plugins
, 
languages
 через CDP

Безопасные лимиты

✅ Безопасно:    1 запрос / 15–30 секунд с одного IP
⚠️  Осторожно:  1 запрос / 5–10 секунд
❌ Блокировка:  1 запрос / 1–3 секунды (устойчивый паттерн)

Рекомендуемый максимум: 50 запросов / час с одного IP

✅ Что работает / что нет

Работает

✅ Поиск позиции товара по запросу и SKU
✅ Пагинация — проверяет до 3 страниц (~100 товаров)
✅ Возвращает
"not_found"
если товар не найден в топ-100
✅ Обход базовой bot-detection Ozon
✅ Retry при временных сбоях (3 попытки, exponential backoff)
✅ Сохранение результатов в JSON файлы
✅ Тест устойчивости — 3 запуска × 30 секунд интервал
✅ Подробное логирование в файл и консоль
✅ Юнит-тесты для ключевой логики

Ограничения

⚠️ При активной Cloudflare защите может потребоваться
--no-headless
⚠️ Ozon периодически меняет структуру страницы — может потребоваться обновление селекторов
⚠️ Без прокси при частых запусках возможна временная блокировка IP
❌ Не решает капчу автоматически — при появлении капчи запуск прерывается


🔧 Конфигурация (.env)
```
# Задержки между запросами (секунды)
## MIN_DELAY=3
## MAX_DELAY=7

# Максимум страниц для проверки (1 страница примерно 36 товаров)
## MAX_PAGES=3

# Товаров на странице Ozon
## ITEMS_PER_PAGE=36

# Скрытый режим браузера (true/false)
HEADLESS=true

# Папки для результатов и логов
RESULTS_DIR=results
LOGS_DIR=logs
```
